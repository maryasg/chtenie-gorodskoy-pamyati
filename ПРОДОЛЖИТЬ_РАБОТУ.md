## Рабочий процесс (привычка)

1. **Archiview:** разметка → «Сохранить разметку и перейти к результату»
2. **`copy_to_website.bat`** (в папке v15) — копирует в `public/explorer/MOSCOW_003/`:
   - `07_...png` → `marked-facade.png`
   - `annotations/manual_annotations.json` → `annotations.json`
   - из `project_v8.json` → `facade-project.json` (матрица для подсветки на фото)
3. **GitHub Desktop:** Commit → **Push origin**
4. **Actions** → Deploy to GitHub Pages — зелёная галочка (2–3 мин)
5. Сайт: **Ctrl+F5** на https://maryasg.github.io/chtenie-gorodskoy-pamyati/

Локально сайт: `npm install` → `npm run dev` (только на вашем ПК).

## Известные нюансы

- **Прямая ссылка на здание** (`/building/MOSCOW_003_dom_so_zveryami`) — после deploy с `404.html` (уже в `vite.config.ts`). Если 404 — зайти с главной или дождаться зелёного Actions и Ctrl+F5.
- **Сайт не обновился** → был Push? Actions зелёный? **Ctrl+F5** (не обычный F5).
- **npm** на ПК может быть не в PATH — для публикации достаточно Push; сборка в GitHub Actions.
- Старые папки v11–v14 на рабочем столе можно не трогать; работать из **v15**.
- Archiview на сайте — только **экспортированные файлы**; правки GUI — в папке v15 на Desktop, затем `copy_to_website.bat`.

### Если что-то сломалось

| Симптом | Что проверить |
|---------|----------------|
| GitHub Desktop не пушит | Remote = `maryasg/chtenie-gorodskoy-pamyati`, не шаблон `ВАШ_ЛОГИН` |
| Сайт старый | Commit + Push; Actions → Deploy — success |
| Страница 404 на github.io | Settings → Pages → Source = **GitHub Actions**; Re-run workflow |
| `/building/...` не открывается | Нужен deploy с плагином `404.html` (уже в репо) |
| На карте лишний текст | Должно быть только © OpenStreetMap (без Leaflet) |
| Нет фото Archiview на карточке | Запустить `copy_to_website.bat`, снова Commit + Push |

## Как начать новый чат

1. Откройте проект `chtenie-gorodskoy-pamyati` в Cursor.
2. **Новый чат** (не продолжайте очень длинный — при ~10% контекста качество падает).
3. Прикрепите **@ПРОДОЛЖИТЬ_РАБОТУ.md** + скрин, если есть баг.
4. Одна конкретная задача, например:
   - «Доработай дизайн карточки MOSCOW_003»
   - «Добавь второе здание с Archiview»
   - «Улучши Archiview v15: …»
5. В конце сессии можно снова попросить: **«обнови файл»** — ассистент допишет этот handoff.
6. Просите **не ломать** карту, экскурсию и остальные здания; **Commit** — только когда вы явно попросите.

Не копируйте всю старую переписку — достаточно этого файла и формулировки задачи.
