# Как работает папка `archiview_projects`

Краткая инструкция: **куда попадают фото и зачем там разные папки**.

---

## Главное правило

**Архив PastVu и все ваши снимки храните где угодно на диске.**  
В `archiview_projects` программа кладёт только **одну выбранную пару** «история + сегодня» для конкретного дома.

---

## Пошагово для нового дома

1. Запустите **`run_gui_windows.bat`** в папке v15 на рабочем столе.
2. Вкладка **«1. Выбор фото»**:
   - Поле **адрес** — например `Москва, Большая Ордынка, 17` (программа назовёт папку по адресу).
   - **Историческое:** поиск PastVu **или** «Файл…» — выберите **одно** подходящее фото.
   - **Современное:** поиск Wikimedia/OSM **или** «Файл…» — **ваша** съёмка с телефона/камеры.
3. Программа создаст папку, например:
   ```
   archiview_projects\Большая_Ордынка_17\
     historical_sources\   ← сюда скопировано историческое
     modern_sources\       ← сюда скопировано современное
     result\               ← сюда попадёт выпрямление и разметка
   ```
4. Вкладки 2–3: углы фасада → выпрямление → разметка → «Сохранить».
5. **`copy_to_website.bat`** — копирует `result/` на сайт в `public/explorer/<ID>/`.
6. GitHub Desktop → Commit → Push.

---

## Почему вы видите странные имена папок

| Папка | Почему так |
|-------|------------|
| `new_house_project` | Первый запуск, дом ещё не назван |
| `20260520_190036` | Старый проект: имя = дата и время создания (создала программа) |
| Имя по адресу | После ввода адреса Archiview сам создаст папку |
| Имя как у файла фото | Если адрес пустой — берётся имя вашего современного файла |

**Не нужно** заранее создавать папки с номерами. Достаточно выбрать фото во вкладке 1.

---

## Что НЕ нужно делать

- ❌ Копировать в `archiview_projects` весь архив или все свои фото
- ❌ Вручную создавать папки `20260520_…` — это делает программа
- ❌ Класть в `result/` файлы руками до выпрямления

## Что нужно делать

- ✅ Хранить архив отдельно (PastVu, PDF экспертиз, ваши RAW/JPG)
- ✅ В Archiview выбирать **лучшую пару** под один ракурс фасада
- ✅ После разметки — `copy_to_website.bat` и Push

## Связь с сайтом

| На ПК (Archiview) | На сайте |
|-------------------|----------|
| `result/03_historical_rectified.png` | `public/explorer/MOSCOW_003/historical-rectified.png` |
| `result/04_modern_rectified.png` | `modern-rectified.png` |
| `result/06_marked_rectified.png` | `marked-facade.png` для overlay |
| `result/10_side_by_side_marked.png` | `side-by-side-marked.png` для side_by_side |
| `annotations/manual_annotations.json` | `annotations.json` |

`07_marked_on_original_modern.png` остаётся полезным для проверки и архива, но для overlay на сайте используется `06_marked_rectified.png`, чтобы координаты полигонов не съезжали.

Для каждого нового здания — свой ID (`MOSCOW_001`, `MOSCOW_002` …) и строка в `src/data/explorer/archiviewAssets.ts`.

## PDF экспертиз (mos.ru)

- файл `src/data/buildings/moscow00X.ts` → блок `verification` и `sources`
