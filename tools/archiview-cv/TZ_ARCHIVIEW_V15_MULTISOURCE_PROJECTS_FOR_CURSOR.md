# ТЗ для разработчика / Cursor
## Archiview CV v15
### Тема: многослойные исторические источники, портрет здания, база обработанных домов, повторное редактирование и подготовка к обновлению сайта

---

## 0. Контекст проекта

Archiview CV — настольное приложение на Python/OpenCV для работы с историческими и современными фотографиями зданий.

Текущий workflow приложения:
1. Пользователь выбирает историческое и современное фото.
2. Выпрямляет фасады по 4 точкам.
3. Настраивает overlay / до-после.
4. Размечает изменения: надстройка, пристройка, заложенное окно, новый балкон и т.д.
5. Получает результат на исходном современном фасаде.
6. Экспортирует разметку для дальнейшей работы / Roboflow / сайта.

В версии v15 нужно перейти от модели:

```text
1 современное фото + 1 историческое фото
```

к модели:

```text
1 проект дома
+ 1 или несколько современных фото
+ много исторических фото
+ несколько сравнений
+ общий портрет здания
+ история правок
+ повторное редактирование
+ подготовка к публикации / обновлению сайта
```

---

## 1. Главная цель v15

Сделать Archiview не разовой программой для одной пары фотографий, а **менеджером проектов по домам**.

Пользователь должен видеть:
- какие дома уже обрабатывались;
- какие исторические фото уже были использованы;
- какие сравнения уже сделаны;
- какие разметки уже сохранены;
- какие элементы уже опубликованы / готовы к сайту;
- какие источники ещё нужно добавить;
- какие результаты можно открыть и поправить.

Главная проблема, которую нужно решить:

> Сейчас можно взять хороший уже найденный исторический ракурс, добавить новое фото, сделать новое сравнение — и старый результат может потеряться или быть перезаписан.

В v15 **ничего не должно теряться**. Каждое сравнение и каждая разметка должны сохраняться как отдельная версия/сессия.

## 3. Основные сущности

### 3.1. HouseProject

Дом / объект исследования.

Поля:

```json
{
  "project_id": "chistoprudnyy_bulvar_14_str_3",
  "address": "Чистопрудный бульвар, д. 14, стр. 3",
  "lat": 55.0,
  "lon": 37.0,
  "object_name": "Доходный дом",
  "heritage_status": "выявленный / ОКН / не указан",
  "act_urls": [],
  "notes": "",
  "created_at": "",
  "updated_at": ""
}
```

### 3.2. PhotoSource

Любое исходное фото: историческое или современное.

Поля:

```json
{
  "photo_id": "hist_001",
  "kind": "historical",
  "title": "PastVu photo 12345",
  "date_from": 1910,
  "date_to": 1915,
  "source_type": "pastvu / file / commons / clipboard / other",
  "source_url": "",
  "license": "",
  "author": "",
  "original_path": "sources/historical/hist_001/original.jpg",
  "preview_path": "sources/historical/hist_001/preview.jpg",
  "rectified_path": "sources/historical/hist_001/rectified.png",
  "points_path": "sources/historical/hist_001/points.json",
  "is_rectified": true,
  "quality_notes": "виден левый фасад, верх обрезан",
  "created_at": "",
  "updated_at": ""
}
```

### 3.3. ComparisonSession

Отдельное сравнение. Не перезаписывать старые сравнения.

Поля:

```json
{
  "comparison_id": "cmp_001",
  "title": "Сравнение с фото PastVu 1910",
  "modern_photo_id": "modern_001",
  "historical_photo_ids": ["hist_001", "hist_002"],
  "active_historical_photo_id": "hist_001",
  "overlay_settings": {},
  "annotation_count": 5,
  "status": "draft / reviewed / ready_for_site / published",
  "created_at": "",
  "updated_at": ""
}
```

### 3.4. Annotation

Область изменения.

Поля:

```json
{
  "annotation_id": "ann_001",
  "number": 1,
  "class_name": "Новый балкон",
  "comment": "Устроен новый балкон над входом",
  "polygon_rectified": [[100, 200], [150, 200], [160, 260], [95, 255]],

  "evidence_historical_photo_ids": ["hist_001", "hist_003"],
  "confidence": "confirmed / probable / check_manually",
  "created_at": "",
  "updated_at": ""
}
```

## 4. Новая вкладка: «База домов / Проекты»

Добавить или расширить вкладку базы домов.

### 4.1. Задачи вкладки

Пользователь должен видеть список всех домов, которые уже есть в проекте.

Для каждого дома в списке показать:
- адрес;
- статус;
- количество исторических фото;
- количество современных фото;
- количество сравнений;
- есть ли опубликованный результат;
- дата последней правки.

Пример таблицы:

```text
Адрес | Ист. фото | Совр. фото | Сравнения | Статус | Последняя правка
Чистопрудный, 14, стр. 3 | 4 | 2 | 3 | Размечается | 2026-06-03
Большая Ордынка, 17, стр. 1 | 2 | 1 | 1 | Готово к сайту | 2026-06-01
```

### 4.2. Функции вкладки

Кнопки:

```text
Создать новый дом
Открыть выбранный дом
Импортировать Excel / CSV
Обновить список
Открыть папку проекта
Экспортировать для сайта
```

Фильтры:

```text
Все
В работе
Нужны фото
Нужно выпрямление
Есть разметка
Готово к публикации
Опубликовано
```

### 4.3. Важное поведение

При открытии дома программа должна загрузить:
- все исторические источники;
- все современные источники;
- все сравнения;
- последнюю активную сессию сравнения;
- разметку, если она есть.

Пользователь должен иметь возможность выбрать существующее сравнение и поправить его, не создавая хаос.

## 5. Много исторических фото

### 5.1. Новая логика

Современное фото становится базовой опорой.
Исторических фото может быть много.

Пользователь может:
- добавить историческое фото из PastVu;
- добавить историческое фото из файла;
- добавить несколько исторических фото подряд;
- для каждого исторического фото отдельно указать точки выпрямления;
- выровнять каждое историческое фото относительно выбранного современного фасада;
- включать/выключать исторические слои.

### 5.2. В интерфейсе

Во вкладке источников добавить блок:

```text
Исторические фото проекта
[+] Добавить из PastVu
[+] Добавить файл
[Удалить из проекта]
[Открыть источник]
```

Таблица:

```text
ID | Год | Источник | Выпрямлено | Использовалось в сравнениях | Комментарий
hist_001 | 1910 | PastVu | Да | cmp_001, cmp_003 | виден левый фасад
hist_002 | 1935 | файл | Да | cmp_002 | виден верх
```

### 5.3. Защита от потери результата

Если пользователь выбирает новое историческое фото, программа НЕ должна перезаписывать старое сравнение.

Нужно предложить:

```text
Создать новое сравнение
Добавить это фото как слой к текущему сравнению
Заменить активный слой, но сохранить старое сравнение как версию
```

По умолчанию безопасный вариант:

```text
Добавить как новый исторический слой
```

## 6. Много современных фото и современная склейка

### 6.1. Зачем это нужно

Иногда современное фото не охватывает весь фасад:
- с одной точки видна левая часть;
- с другой — правая;
- с третьей — верх или боковая пристройка.

Нужно предусмотреть возможность собрать современный фасад из нескольких современных фото.

### 6.2. Минимальный вариант v15

В v15 не обязательно делать полностью автоматическую фотограмметрию.
Сделать практичный полуавтоматический режим:

```text
1. Пользователь добавляет несколько современных фото.
2. Для каждого фото указывает 4 точки той же фасадной плоскости.
3. Программа выпрямляет каждое фото в общую плоскость.
4. Пользователь включает/выключает слои.
5. Программа может создать современный композит фасада.
```

### 6.3. Современный композит

Добавить вкладку или режим:

```text
Современный композит
```

Функции:
- список современных слоёв;
- прозрачность каждого слоя;
- порядок слоёв;
- вручную выбрать область, которую взять из конкретного фото;
- собрать `modern_composite.png`.

Файлы:

```text
composites/modern_portrait_001/composite.png
composites/modern_portrait_001/composite_layers.json
```

### 6.4. Приоритет

На первом этапе достаточно:
- хранить несколько современных фото;
- выпрямлять каждое;
- выбирать главное современное фото для сравнения;
- предусмотреть структуру и UI под будущую склейку.

Если успеете — добавить простой ручной композит.

## 7. Портрет здания из исторических фрагментов

### 7.1. Цель

Собрать единый исторический образ фасада из нескольких старых фото.

Например:
- верх фасада из hist_002;
- левый фрагмент из hist_001;
- балконы из hist_003;
- декор из hist_004.

### 7.2. UI

Добавить вкладку:

```text
Исторические слои / Портрет здания
```

Слева:
- список исторических фото;
- чекбокс видимости;
- прозрачность;
- комментарий;
- год / источник.

Справа:
- общий выпрямленный фасад;
- режимы:
  - показать одно фото;
  - overlay с современным;
  - собрать исторический портрет;
  - разметить фрагмент источника.

### 7.3. Ручная сборка композита

Пользователь должен иметь возможность нарисовать полигон и указать:

```text
Эту область взять из hist_002
```

Программа сохраняет:

```json
{
  "composite_id": "historical_portrait_001",
  "fragments": [
    {
      "source_photo_id": "hist_002",
      "polygon_rectified": [[...]],
      "comment": "верх фасада лучше виден здесь"
    }
  ]
}
```

На выходе:

```text
historical_portrait_001/composite.png
historical_portrait_001/composite_layers.json
```

## 8. Сравнения: список и история

### 8.1. Новая вкладка / блок «Сравнения»

Пользователь должен видеть все сравнения по дому:

```text
ID | Современное фото | Исторические фото | Разметок | Статус | Дата
cmp_001 | modern_001 | hist_001 | 5 | Готово | 2026-06-01
cmp_002 | modern_001 | hist_002, hist_003 | 8 | В работе | 2026-06-03
```

### 8.2. Функции

Кнопки:

```text
Открыть сравнение
Создать новое сравнение
Дублировать сравнение
Добавить исторический слой
Добавить современный слой
Архивировать сравнение
```

### 8.3. Безопасность

Никогда не перезаписывать `cmp_001`, если пользователь создаёт новый вариант.
Если пользователь хочет изменить старую разметку:
- открыть существующее сравнение;
- внести правки;
- сохранить как новую версию или обновить текущую после подтверждения.

Добавить диалог:

```text
Вы редактируете существующее сравнение cmp_001.
Сохранить изменения в нём или создать копию cmp_004?
```

По умолчанию безопасный вариант:

```text
Создать копию
```

## 10. Синхронизация / обновление сайта

### 10.1. В v15 сделать подготовку к сайту

Не обязательно делать реальную публикацию на сайт, если нет API сайта.
Но нужно сделать **стабильный экспорт для сайта**.

Папка:

```text
exports/site/
  house_card.json
  comparison_cmp_001.json
  annotations.json
  images/
    modern_result.png
    modern_result_labeled.png
    overlay_preview.png
    before_after_preview.png
    historical_sources/
```

### 10.2. Статусы публикации

У дома и у сравнения должны быть статусы:

```text
draft — черновик
reviewed — проверено
ready_for_site — готово к сайту
published — опубликовано
needs_update — нужно обновить
```

Если пользователь правит уже опубликованный дом, статус должен стать:

```text
needs_update
```

### 10.3. Экспорт изменений

Добавить кнопку:

```text
Экспортировать обновление для сайта
```

Она создаёт:

```text
exports/site_update_YYYYMMDD_HHMM.zip
```

Внутри:
- JSON с данными дома;
- JSON с актуальной разметкой;
- изображения результата;
- список источников;
- статус.

### 10.4. Если сайт имеет API в будущем

Предусмотреть интерфейс:

```text
Настройки сайта:
API URL
API token
```

Но пока можно оставить как заготовку / TODO.

## 12. Рекомендованный новый workflow

### 12.1. Открытие существующего дома

```text
1. Вкладка “База домов”.
2. Выбрать дом.
3. Увидеть список уже добавленных фото.
4. Увидеть список уже созданных сравнений.
5. Открыть нужное сравнение или создать новое.
```

### 12.2. Добавление нового исторического фото

```text
1. Открыть дом.
2. Добавить историческое фото из PastVu / файла.
3. Фото сохраняется как hist_00N.
4. Пользователь выпрямляет его к выбранному современному фото.
5. Программа предлагает:
   — добавить в текущее сравнение;
   — создать новое сравнение;
   — оставить как источник без сравнения.
```

### 12.3. Разметка с несколькими историческими слоями

```text
1. Открыть сравнение.
2. Включить один или несколько исторических слоёв.
3. Настроить overlay.
4. Разметить изменения.
5. Для каждого изменения указать источник подтверждения.
6. Сохранить.
7. Экспортировать для сайта.
```

## 15. Критерии приёмки

Версия v15 считается принятой, если:

1. В одном доме можно добавить минимум 3 исторических фото.
2. Для каждого исторического фото можно сохранить исходник, точки и выпрямленную версию.
3. Можно создать минимум 2 разных сравнения по одному дому.
4. Старое сравнение не пропадает при добавлении нового исторического фото.
5. В интерфейсе видно, какие фото уже использовались в каких сравнениях.
6. Можно открыть старое сравнение и поправить разметку.
7. У каждой области разметки можно указать исторический источник подтверждения.
8. Можно экспортировать папку `exports/site/` или ZIP для сайта.
9. Структура проекта не ломает старые папки предыдущих версий.

## 16. Короткое резюме задачи для Cursor

Сделай Archiview CV v15 как систему проектов по домам:
- один дом может иметь много исторических и современных фото;
